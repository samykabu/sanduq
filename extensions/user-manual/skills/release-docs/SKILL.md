# Release documentation

Read [release-notes.md](references/release-notes.md) for any release-visible change. Read
[migration-guides.md](references/migration-guides.md) only when readers must take action.

Ground every item in feature artifacts and delivered changes. Separate end-user improvements,
administrator actions, and technical compatibility details. Use plain language and state who is
affected, what changed, when action is required, and how to verify success.

Never invent versions, dates, issue numbers, support windows, or compatibility guarantees.
