# Screenshot capture matrix

For each applicable feature, cover the smallest useful combination of:

- Audience/role: end user, delegated user, administrator, operator, denied role.
- Interface: desktop web, responsive web, and real native mobile where implemented.
- State: happy path, empty, loading, validation, error, permission denied, and meaningful completion.
- Language/direction: English LTR and Arabic RTL when Arabic is enabled.
- Theme: default plus alternate theme only when it materially changes comprehension.

Do not create a Cartesian explosion. Select states that teach a user or prove a documented branch.
Run captures in a pinned environment because browser, OS, font, and rendering differences make
screenshots unstable across machines.
