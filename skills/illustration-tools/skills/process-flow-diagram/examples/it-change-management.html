IT Change Management

ITIL-aligned change request through CAB approval, implementation, and post-implementation review

Prerequisites

  • • ServiceNow (or equivalent) configured
  • • CAB roster + cadence agreed
  • • CMDB current and reachable
  • • Risk matrix & change policy published
  • • Rollback runbooks per service

Inputs → Outputs

  • • Input: Request for Change (RFC)
  • • Input: CMDB blast-radius graph
  • • Output: CAB recommendation
  • • Output: Scheduled change window
  • • Output: PIR + closed change record

Tools & Integrations

  • • ServiceNow (records, scheduling)
  • • CMDB (impact + dependency graph)
  • • Calendar (change windows)
  • • Slack / email (CAB & stakeholder notifications)
