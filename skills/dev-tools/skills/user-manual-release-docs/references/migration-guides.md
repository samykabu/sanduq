# Migration guides

Create a guide when a release changes data, API contracts, configuration, permissions, workflows,
deployment, compatibility, or user behavior in a way that requires action.

Include affected audience and versions, prerequisites, backup/safety checks, before/after behavior,
ordered migration steps, verification, rollback, downtime or compatibility constraints, and known
limitations. For database work, describe schema operations without exposing production records or
credentials. For APIs, include old/new contract examples and deprecation timing only when evidenced.
