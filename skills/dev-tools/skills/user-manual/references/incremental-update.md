# Incremental update

Map the active feature to approved modules and page types before editing. Use spec stories, tasks,
contracts, changed files, tests, routes, schema changes, and observed UI behavior as evidence.

Update only affected pages and shared indexes. Preserve unaffected content and historical releases.
Refresh screenshots or diagrams when the visible state or documented relationship changed. Update
coverage metadata with each page's evidence and verification date.

When a feature introduces an unrecognized product area, add a proposed module with evidence and ask
for approval before stable navigation. A missing full baseline triggers initialization; never create
a disconnected feature page as a substitute.
