# Audience and language

## End User

Explain what the feature helps the person accomplish. Use familiar product words, short steps,
visible labels, expected outcomes, and common recovery guidance. Omit architecture and internal IDs.

## Administrator or Operator

Explain configuration, roles, permissions, operational effects, monitoring, audit behavior,
recovery, dependencies, and safe rollback. Define technical terms on first use.

## Technical Reference

Use exact names and types where precision matters. Document interfaces, contracts, components,
infrastructure, security boundaries, entities, columns, constraints, enumerations, and evidence
paths. Still prefer direct sentences over unexplained jargon.

## English and Arabic

English is complete and required. Arabic is optional by project policy. Write natural Arabic for the
reader rather than word-for-word translation. Preserve code, endpoint, field, and product names when
translation would make them inaccurate. Add `dir="rtl"` behavior through the theme and verify mixed
Arabic/Latin text, tables, callouts, diagrams, code, and PDF pagination.
